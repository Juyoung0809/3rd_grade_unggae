## BUG-001: User.name → User.nickname ERD 불일치

### 발생일
2026-04-20

### 증상
기존 코드에서 `User.name` 필드를 사용하고 있었으나 ERD 명세에는 `nickname` 컬럼으로 정의되어 있음.
프론트엔드 회원가입 폼에 "이름" 레이블로 표시되고 있었으나 실제 의미는 닉네임.

### 원인
초기 구현 시 ERD와 코드 간 불일치 발생. `User.name`, `User.profileImage` 등 필드명이 docs와 다름.

### 수정 내용
- `User.java`: `name` → `nickname` (`@Column(name = "nickname")`), `profileImage` → `profileImageKey`, `bio` TEXT 필드 추가
- `AuthService`, `RegisterRequest`, `TokenResponse` 등 연쇄 수정
- DTO 9개 파일에서 `getName()` → `getNickname()` 변경
- 프론트엔드 전체 `name` → `nickname` 인터페이스 동기화

### 검증
`./gradlew compileJava` BUILD SUCCESSFUL, `npm run build` 성공 확인
