## 1. 기능 요구사항

### 1.1 인증 (AUTH)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| AUTH-01 | 사용자는 이메일과 비밀번호로 회원가입할 수 있다. | 필수 |
| AUTH-02 | 회원가입 시 역할(STUDENT / INSTRUCTOR)을 선택한다. | 필수 |
| AUTH-03 | 이메일 중복 시 회원가입을 거부한다. | 필수 |
| AUTH-04 | 로그인 성공 시 Access Token과 Refresh Token을 발급한다. | 필수 |
| AUTH-05 | Access Token 만료 시 Refresh Token으로 재발급 가능하다. | 필수 |
| AUTH-06 | 로그아웃 시 Refresh Token을 무효화한다. | 필수 |
| AUTH-07 | 비밀번호는 BCrypt로 암호화하여 저장한다. | 필수 |

---

### 1.2 사용자 (USER)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| USER-01 | 사용자는 자신의 프로필(닉네임, 소개, 프로필 이미지)을 조회할 수 있다. | 필수 |
| USER-02 | 사용자는 자신의 프로필을 수정할 수 있다. | 필수 |
| USER-03 | 프로필 이미지는 업로드하며, DB에는 이미지 경로를 저장한다. | 필수 |
| USER-04 | 관리자는 특정 사용자의 역할을 변경할 수 있다. | 필수 |
| USER-05 | 관리자는 사용자를 비활성화(정지)할 수 있다. | 필수 |
| USER-06 | 비활성화된 사용자는 로그인이 거부된다. | 필수 |

---

### 1.3 강의 (COURSE)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| COURSE-01 | 강사는 강의를 등록할 수 있다 (제목, 설명, 카테고리, 가격, 썸네일). | 필수 |
| COURSE-02 | 등록된 강의는 기본적으로 PENDING 상태이며, 관리자 승인 후 PUBLISHED 된다. | 필수 |
| COURSE-03 | 강사는 자신의 강의를 수정하거나 삭제할 수 있다. | 필수 |
| COURSE-04 | 삭제는 실제 삭제 대신 비활성화(soft delete)로 처리한다. | 필수 |
| COURSE-05 | 수강생/비회원은 공개된 강의 목록과 상세를 조회할 수 있다. | 필수 |
| COURSE-06 | 강의 목록은 카테고리, 키워드, 정렬(최신순/평점순/수강생순)로 필터링된다. | 필수 |
| COURSE-07 | 강의 상세에는 소개, 커리큘럼, 강사 정보, 평점, 리뷰 목록이 포함된다. | 필수 |

---

### 1.4 섹션 & 레슨 (SECTION / LESSON)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| SEC-01 | 강사는 강의 내에 여러 섹션(챕터)을 추가/수정/삭제할 수 있다. | 필수 |
| SEC-02 | 섹션은 순서(order)를 가지며, 순서 변경이 가능하다. | 필수 |
| LES-01 | 강사는 섹션 내에 레슨(강의 영상)을 추가/수정/삭제할 수 있다. | 필수 |
| LES-02 | 레슨은 제목, 설명, 영상 URL, 영상 길이(초)를 가진다. | 필수 |
| LES-04 | 레슨도 순서(order)를 가지며, 순서 변경이 가능하다. | 필수 |
| LES-05 | 미리보기 레슨은 비수강생도 조회 가능하다. | 선택 |

---

### 1.5 수강 신청 & 진도 (ENROLLMENT / PROGRESS)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| ENROLL-01 | 수강생은 무료 강의를 즉시 수강 신청할 수 있다. | 필수 |
| ENROLL-02 | 이미 수강 중인 강의는 중복 신청이 불가하다. | 필수 |
| ENROLL-03 | 수강생은 자신의 수강 목록을 조회할 수 있다. | 필수 |
| PROG-01 | 수강생은 레슨 시청 완료 시 진도를 업데이트할 수 있다. | 필수 |
| PROG-02 | 강의 전체 진도율은 (완료 레슨 수 / 전체 레슨 수)로 계산된다. | 필수 |
| PROG-03 | 수강생은 강의별 전체 진도율을 조회할 수 있다. | 필수 |

---

### 1.6 리뷰 (REVIEW)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| REVIEW-01 | 수강 중인 수강생만 리뷰를 작성할 수 있다. | 필수 |
| REVIEW-02 | 리뷰는 별점(1~5)과 내용을 포함한다. | 필수 |
| REVIEW-03 | 수강생은 자신의 리뷰를 수정 및 삭제할 수 있다. | 필수 |
| REVIEW-04 | 강의당 1인 1리뷰만 허용한다. | 필수 |
| REVIEW-05 | 강의 평균 별점은 리뷰 등록/수정/삭제 시 자동으로 갱신된다. | 필수 |

---

### 1.7 결제 내역 (PAYMENT)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| PAY-08 | 수강생은 자신의 결제 내역을 조회할 수 있다. | 필수 |

---

### 1.8 관리자 (ADMIN)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| ADMIN-01 | 관리자는 전체 회원 목록을 조회하고 상태를 변경할 수 있다. | 필수 |
| ADMIN-02 | 관리자는 PENDING 상태 강의를 승인(PUBLISHED)하거나 반려(REJECTED)할 수 있다. | 필수 |
| ADMIN-03 | 관리자는 전체 결제 내역을 조회하고 환불 처리를 할 수 있다. | 필수 |
| ADMIN-04 | 관리자는 서비스 주요 통계(총 회원수, 총 매출, 강의수 등)를 조회할 수 있다. | 선택 |

## 2. 공통 응답 포맷

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

오류 응답:
```json
{
  "code": 400,
  "message": "이미 수강 중인 강의입니다.",
  "data": null
}
```
